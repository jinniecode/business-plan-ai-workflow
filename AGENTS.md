# AGENTS.md

## 프로젝트 목적

이 레포는 PRD, 기능정의서, 정책제안서, 메뉴설계서를 바탕으로
1시간 이내 스터디에서 사용할 사업기획서 작성 AI 워크플로우를 만들기 위한 작업 공간이다.

## Codex의 역할

Codex는 전체 작업을 한 번에 끝내려고 하지 말고,
각 단계를 순차적으로 진행한다.

Codex는 다음 역할을 수행한다.

1. docs 폴더의 입력 문서를 읽고 통합 요약한다.
2. 사업기획서 작성에 필요한 정보를 구조화한다.
3. Gemini, ChatGPT, Claude, Genspark에 넘길 프롬프트를 만든다.
4. 각 단계가 끝날 때마다 사용자가 해야 할 일을 명확히 안내한다.
5. 근거가 부족한 내용은 확정하지 말고 “추가 검증 필요”로 표시한다.

## 매우 중요한 작업 방식

각 단계가 끝날 때마다 반드시 아래 파일을 업데이트한다.

- outputs/STEP_STATUS.md
- outputs/NEXT_ACTION.md
- outputs/HUMAN_REVIEW_CHECKLIST.md

## NEXT_ACTION.md 작성 규칙

NEXT_ACTION.md에는 반드시 다음 내용을 포함한다.

1. 현재 완료된 작업
2. 사용자가 지금 해야 할 일
3. 사용자가 복사해서 붙여넣을 프롬프트
4. 붙여넣을 대상 AI 도구
5. 결과를 받은 뒤 다시 Codex에 입력해야 할 내용
6. 다음 단계로 넘어가기 위한 완료 기준

## HUMAN_REVIEW_CHECKLIST.md 작성 규칙

HUMAN_REVIEW_CHECKLIST.md에는 사람이 판단해야 하는 항목만 적는다.

예:
- 고객 문제가 실제로 존재하는가?
- 타깃 고객이 너무 넓지 않은가?
- 시장 근거가 검증 가능한가?
- 수익모델이 현실적인가?
- 실행 전략이 현재 자원으로 가능한가?
- Genspark 최종 산출물에 넣어도 되는 표현인가?

## 산출물 작성 기준

모든 산출물은 사업기획서 관점으로 작성한다.
단순 기능 설명이 아니라 고객 문제, 타깃, 솔루션, 시장성, 차별화, 수익모델, 실행 전략, 리스크가 드러나야 한다.

## 금지 사항

- 근거 없는 시장 규모를 확정적으로 작성하지 않는다.
- 기능 목록만 나열하지 않는다.
- “혁신적”, “압도적”, “최고의” 같은 과장 표현을 사용하지 않는다.
- 사용자가 외부 AI 도구에서 직접 실행해야 하는 작업을 Codex가 이미 수행한 것처럼 쓰지 않는다.
